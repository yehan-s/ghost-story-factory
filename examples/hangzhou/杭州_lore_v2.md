## 第五部分：核心游戏系统（Core Gameplay Systems）

### 5.1 共鸣机制正式化
| 维度 | 范围 | 初始值 | 计算方式 | 影响边界 |
|---|---|---|---|---|
| 全局共鸣度（Global Resonance, GR） | 整个服务器/存档 | 0 | 所有玩家个人共鸣度加权平均 + 世界事件修正 | 0–100；每+10GR，B3层刷新率+5%，实体T0→T1门槛-2 |
| 个人共鸣度（Personal Resonance, PR） | 单个角色 | 0 | 完成交互、成功解密、击杀或安抚实体 | 0–100；死亡不掉落，但“记忆缺口”会随机-10PR |

### 5.2 Linus 式统一映射（PR ↔ 实体行为等级）
```
PR 0–19  → 所有实体停留在 T0
PR 20–39 → 可触发 T1
PR 40–59 → 可触发 T2
PR 60–79 → 可触发 T3
PR 80–100→ 可触发 T4
```
> 说明：个人共鸣度决定“你能看见的世界深度”；全局共鸣度决定“世界有多活跃”。两者独立计算，但高GR会缩短实体刷新CD，间接加速PR上升。

### 5.3 世界疲劳值（World Fatigue, WF）
* 每次使用“雨夜东风模拟器”+1WF
* WF上限=10；每+1WF，全局共鸣度自然衰减速度+5%（即服务器每日自动-GR）
* WF=10时，触发“永久静默”事件：所有65Hz信号消失，世界进入“无声模式”，需全服玩家共同完成“核心解密”才能重置WF为0

### 5.4 可重玩性设计
1. 角色身份差异：8种初始职业（1.0 allowed_roles）各自携带不同“监听”天赋，如“录音设备测评博主”初始+15PR，“夜班保安”可自由进出值班室。
2. 动态气象接口：游戏实时读取杭州真实天气API，雨停+东风3-4级条件天然不可控，迫使玩家利用“雨夜东风模拟器”或等待，形成天然节奏差。
3. 多结局分支：
   * “第三节车厢”真相结局：需GR≥80、PR≥80、持有【失魂核心】，在数字孪生车厢内播放0:18录音，世界进入“补完”状态，全体玩家获得“记忆完整”称号，服务器重置WF。
   * “永久静默”结局：WF=10且30天内未重置，世界关闭65Hz通道，所有实体冻结，玩家只能以“旁观者”模式继续探索，形成“无声博物馆”永久展览。
   * “夺魂蔓延”结局：玩家失魂者数量>在线玩家50%，服务器自动开启“夺魂入侵”事件，新玩家默认PR=0但外观为失魂者，需反向解密“找回自己”。
